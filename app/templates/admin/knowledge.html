▶ How knowledge scoping works (channel, bot, global)

Knowledge documents are scoped via knowledge_access entries. Each entry defines a scope (channel, bot, or global) and a mode (rag, pinned, or tag_only).

Scope types

  • Channel — Visible only in a specific channel. Knowledge persists across session resets.
  • Bot — Visible to a specific bot across all channels.
  • Global — Visible to all bots in all channels.

Retrieval modes

  • RAG — Participates in automatic semantic similarity search. Retrieved when the query is relevant.
  • Pinned — Always injected into context, regardless of similarity.
  • Tag only — Only accessible via explicit @document_name reference. Never auto-retrieved.

@name references

Explicit @document_name references work for all modes — even tag_only documents. The reference resolves by name within the accessible scope.
