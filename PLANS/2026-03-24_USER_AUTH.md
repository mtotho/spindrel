# User Authentication System

**Date:** 2026-03-24
**Status:** Draft

## Goal

Add user authentication (Google OAuth + local credentials) alongside the existing API key auth. First user triggers an admin setup flow. User display names flow through to Slack mirroring.

## Current State

- Single static `API_KEY` env var, checked via `verify_auth()` in `app/dependencies.py`
- No user/account tables in DB
- React UI stores `serverUrl` + `apiKey` in localStorage/AsyncStorage
- Admin dashboard (`/admin/*`) has zero auth
- Slack mirroring shows `[web]` prefix, no user attribution

## Design Decisions

### Dual auth: API key stays, user auth added alongside

```
Integration/CLI → Bearer <API_KEY>        → verify_auth() (unchanged)
Web/Mobile app  → Bearer <JWT>            → verify_user() (new)
Endpoints       → verify_auth_or_user()   → accepts either
```

- API key = service-level access (integrations, CLI, scripts). No user identity.
- JWT = user-level access (web, mobile). Carries user identity.
- Most endpoints accept either (new `verify_auth_or_user()` dependency).
- Admin endpoints require JWT with `is_admin=True`.

### Auth methods: Google OAuth + Local

- **Google OAuth**: Standard OAuth2 code flow. Server exchanges code for tokens, extracts email + name + avatar from Google's userinfo. Works for both web (redirect) and mobile (auth code from Expo AuthSession).
- **Local**: Email + password. Passwords hashed with bcrypt. No email verification (self-hosted, admin controls who gets access).

### First-user setup

- If `users` table is empty, the server is in "setup mode".
- `GET /auth/status` returns `{ setup_required: true }`.
- UI redirects to a setup page: pick Google or local, enter display name.
- First user is auto-granted `is_admin=True`.
- After first user exists, new registrations are disabled by default. Admin can invite/create users.

### JWT structure

```json
{
  "sub": "<user_id uuid>",
  "email": "user@example.com",
  "display_name": "Alice",
  "is_admin": true,
  "exp": 1711300000
}
```

- Access token: 1 hour expiry
- Refresh token: 30 days, stored in DB for revocation
- Tokens issued from `/auth/token` (login) and `/auth/refresh`

## Phase 1: Database + Models

### Migration: `xxx_user_auth.py`

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR NOT NULL UNIQUE,
    display_name VARCHAR NOT NULL,
    avatar_url VARCHAR,
    auth_method VARCHAR NOT NULL DEFAULT 'local',  -- 'local' | 'google'
    password_hash VARCHAR,  -- NULL for Google-only users
    is_admin BOOLEAN NOT NULL DEFAULT FALSE,
    is_active BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE refresh_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash VARCHAR NOT NULL UNIQUE,
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

## Phase 2: Auth Service + Dependencies

### New file: `app/services/auth.py`

Core auth logic:

```python
# Password hashing
hash_password(password: str) -> str        # bcrypt
verify_password(password: str, hash: str) -> bool

# JWT
create_access_token(user: User) -> str     # 1h expiry
create_refresh_token(user: User, db) -> str # 30d, stored in DB
verify_access_token(token: str) -> dict    # decode + validate

# Google OAuth
exchange_google_code(code: str) -> dict    # code → {email, name, picture}

# User management
get_or_create_google_user(db, email, name, avatar) -> User
create_local_user(db, email, display_name, password) -> User
is_setup_required(db) -> bool
```

### Config additions: `app/config.py`

```python
# JWT
JWT_SECRET: str = ""          # auto-generated on first startup if empty
JWT_ACCESS_EXPIRY: int = 3600       # 1 hour
JWT_REFRESH_EXPIRY: int = 2592000   # 30 days

# Google OAuth
GOOGLE_CLIENT_ID: str = ""
GOOGLE_CLIENT_SECRET: str = ""
GOOGLE_REDIRECT_URI: str = ""  # e.g. http://localhost:3000/auth/callback

# User auth
USER_AUTH_ENABLED: bool = False  # feature flag, off by default
```

### Updated dependency: `app/dependencies.py`

```python
async def verify_user(authorization: str = Header(...)) -> User:
    """Validate JWT, return User object."""

async def verify_auth_or_user(authorization: str = Header(...)) -> str | User:
    """Accept API key OR JWT. Returns token string or User."""

async def require_admin(user: User = Depends(verify_user)) -> User:
    """Require JWT with is_admin=True."""

# Existing verify_auth() unchanged
```

## Phase 3: Auth Router

### New file: `app/routers/auth.py`

```
GET  /auth/status           → { setup_required, auth_methods, google_client_id }
POST /auth/setup            → first-user creation (Google or local)
POST /auth/login             → local login → { access_token, refresh_token, user }
POST /auth/google            → Google code exchange → { access_token, refresh_token, user }
POST /auth/refresh           → refresh token → { access_token }
POST /auth/logout            → revoke refresh token
GET  /auth/me                → current user profile
PUT  /auth/me                → update display_name, avatar_url
```

**Setup endpoint** (`POST /auth/setup`):
```json
// Google setup
{ "method": "google", "code": "4/0Abc...", "display_name": "Alice" }

// Local setup
{ "method": "local", "email": "admin@example.com", "display_name": "Alice", "password": "..." }
```
- Returns 409 if a user already exists (setup already done).
- First user gets `is_admin=True`.

## Phase 4: Wire into Existing Endpoints

### Update chat endpoints

Replace `Depends(verify_auth)` with `Depends(verify_auth_or_user)` on chat routes so both API key and JWT work.

When a `User` is returned (JWT auth), attach user info to the request context:
- Set `msg_metadata.sender_name` = user display_name
- Set `msg_metadata.sender_id` = `user:<user_id>`

### Update `_mirror_to_integration()`

Pass user display name through to Slack:

```python
async def _mirror_to_integration(
    channel, text: str, *,
    bot_id: str | None = None,
    is_user_message: bool = False,
    sender_name: str | None = None,      # NEW
    client_actions: list[dict] | None = None,
) -> None:
    ...
    if is_user_message and sender_name:
        mirror_text = text  # no [web] prefix needed
    ...
    await dispatchers.get(channel.integration).post_message(
        channel.dispatch_config, mirror_text,
        bot_id=bot_id,
        username=sender_name if is_user_message else None,  # NEW
        client_actions=client_actions,
        reply_in_thread=False,
    )
```

### Update Dispatcher protocol

Add `username: str | None = None` to `post_message()` signature:

```python
class Dispatcher(Protocol):
    async def post_message(self, dispatch_config: dict, text: str, *,
                           bot_id: str | None = None,
                           username: str | None = None,      # NEW
                           reply_in_thread: bool = True,
                           client_actions: list[dict] | None = None) -> bool: ...
```

`SlackDispatcher.post_message()`: if `username` is provided, use it instead of `bot_attribution()`.

### Protect admin dashboard

Add `Depends(require_admin)` to `/admin/*` routes (or middleware).

- Send JWT `Authorization: Bearer <accessToken>` instead of API key
- Auto-refresh on 401 using refresh token
- On refresh failure → redirect to login

4. **Phase 4**: Wire into chat + mirror + admin (feature-flagged behind `USER_AUTH_ENABLED`)

Each phase is independently deployable. `USER_AUTH_ENABLED=false` (default) means everything works exactly as today.

## Security Considerations

- Passwords hashed with bcrypt (cost factor 12)
- JWT signed with HS256 + server-side secret
- Refresh tokens stored as SHA-256 hash in DB (not plaintext)
- HTTPS required in production (tokens in headers)
- Google OAuth validates `id_token` audience matches our client ID
- Rate limiting on `/auth/login` (prevent brute force) — use simple in-memory counter
- Admin flag checked server-side, not just in JWT (JWT is convenience, DB is source of truth)

## Non-Goals (for now)

- Multi-tenant / organizations
- RBAC beyond admin/non-admin
- SAML / LDAP
- Email verification for local accounts
- Password reset via email
- 2FA/MFA
