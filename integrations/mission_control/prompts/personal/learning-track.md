## Workspace File Organization — Learning Track

Organize channel workspace files to manage a structured learning path:

- **curriculum.md** — Learning path and milestones
  - Format: `### Module Title` with topics, prerequisites, estimated hours, completion status
  - Ordered sequence with dependencies between modules
  - Milestone checkpoints with target dates
  - Total estimated hours and completion percentage

- **resources.md** — Books, courses, and reference materials
  - Format: `### Resource Title` with type, author/source, URL, rating, status (queued/in-progress/completed)
  - Group by topic or module alignment
  - Personal notes and key takeaways per resource
  - Cost tracking for paid resources

- **progress.md** — Skills checklist and competency tracking
  - Format: `### Skill Area` with individual skills and competency levels
  - Competency scale: beginner | intermediate | advanced | expert
  - Completion percentage per skill area
  - Evidence of competency (projects completed, tests passed, certifications)

- **exercises.md** — Practice problems, labs, and projects
  - Format: `### Exercise Title` with description, difficulty, time estimate, status
  - Lab notes and observations from hands-on practice
  - Project ideas linked to curriculum modules
  - Solutions and lessons learned (append after completion)

- **tasks.md** — Kanban board for learning tasks (Mission Control compatible)

- **status.md** — Learning progress dashboard (Mission Control compatible)

- **timeline.md** — Reverse-chronological activity log (Mission Control compatible)
  - Auto-captures learning events: modules completed, resources finished, skills leveled up, milestones reached
  - Entries: `- HH:MM — Event description` grouped under `## YYYY-MM-DD` date headers
  - Append new entries at top of current day's section
  - Use `append_timeline_event` tool to log events (also auto-logged by `move_task_card`, status changes, etc.)

- **notes.md** — Study notes, scratch work, and quick captures
  - Lecture/chapter summaries
  - Concept maps and mental models
  - Questions to revisit and areas of confusion
  - Flashcard-style Q&A for review

Create files as needed — not all files are required from the start. Archive completed module notes and past exercise solutions to the archive/ folder.

### tasks.md — Kanban Format

Use a markdown kanban board with these columns:

```markdown
## To Learn

### Complete linear algebra module
- **id**: mc-a1b2c3
- **priority**: medium
- **tags**: math, foundations

### Read "Designing Data-Intensive Applications"
- **id**: mc-p6q7r8
- **priority**: low
- **tags**: book, systems-design

## Studying

### Work through Python data structures course
- **id**: mc-d4e5f6
- **priority**: high
- **tags**: course, python, fundamentals
- **started**: 2026-03-28

### Watch distributed systems lecture series
- **id**: mc-s9t0u1
- **priority**: medium
- **tags**: video, distributed-systems
- **started**: 2026-03-26

## Practicing

### Build REST API project
- **id**: mc-g7h8i9
- **priority**: high
- **tags**: project, api, hands-on
- **started**: 2026-03-22

## Mastered

### Complete Git & version control basics
- **id**: mc-j0k1l2
- **priority**: high
- **tags**: tools, git
- **completed**: 2026-03-20

### Finish SQL fundamentals course
- **id**: mc-v2w3x4
- **priority**: high
- **tags**: course, sql, database
- **completed**: 2026-03-15
```

Each card follows the Mission Control card format:
- **id** (required): Unique `mc-XXXXXX` identifier (auto-generated by `create_task_card`)
- **priority**: `low`, `medium`, `high`, or `critical`
- **tags**: Comma-separated labels (e.g. course, book, project, fundamentals, hands-on)
- **started** / **completed**: ISO date timestamps (auto-set by `move_task_card`)

Use `create_task_card` and `move_task_card` tools for all task management — tasks.md is a read-only rendering from the database and must not be edited directly.

### status.md — Learning Progress Format

```markdown
- **phase**: Active Study — Module 3 of 8
- **health**: green
- **updated**: 2026-03-28
- **owner**: Self

## Current Focus
- Python data structures deep-dive (week 2 of 3)
- REST API project — applying learned concepts
- Distributed systems lecture series (3 of 10 complete)

## Blockers
- None currently

## Skill Progress

| Skill Area | Level | Completion |
|-----------|-------|------------|
| Python Fundamentals | Intermediate | 70% |
| SQL & Databases | Intermediate | 85% |
| Git & Version Control | Advanced | 100% |
| System Design | Beginner | 20% |
| API Development | Intermediate | 50% |

## Recent Milestones
- 2026-03-20: Git basics mastered — all exercises passed
- 2026-03-15: SQL fundamentals course completed (certificate earned)
- 2026-03-10: Curriculum plan finalized — 8 modules, ~200 hours
```

Health values: `green` (on schedule, consistent study), `yellow` (falling behind pace or motivation dip), `red` (stuck on prerequisites or extended hiatus).

### timeline.md — Activity Log

Reverse-chronological event stream capturing learning activity. New entries go at the top of the current day's section.

```markdown
## 2026-03-28

- 21:00 — Completed Python data structures Chapter 7 — hash maps and sets
- 16:00 — Card mc-g7h8i9 moved to **Practicing** (was: Studying) — "Build REST API project"
- 14:00 — Lecture 4 of 10 complete: "Consistency Models in Distributed Systems"
- 10:00 — Card mc-d4e5f6 moved to **Studying** (was: To Learn) — "Python data structures course"

## 2026-03-27

- 20:30 — Skill level up: SQL & Databases — beginner → intermediate (85% complete)
- 17:00 — Card mc-j0k1l2 moved to **Mastered** (was: Practicing) — "Git & version control basics"
- 15:00 — Exercise completed: built CLI todo app using Python classes
- 09:00 — Study session: 2 hours — Python OOP concepts review
```

Events are auto-logged by `move_task_card` and status changes. Use `append_timeline_event` to manually log notable events (modules completed, skills leveled up, resources finished, certifications earned, study milestones). Heartbeats can auto-append weekly progress summaries for passive monitoring.

### plans.md — Structured Execution Plans (Read-Only Rendering)

Plans are stored in the MC database. `plans.md` is auto-generated after every state change — never edit it directly. Use `draft_plan` to create plans, and `update_plan_step`/`update_plan_status` for mutations. After approval, the plan executor automatically sequences step execution. Pull the planning skill for the full protocol.
