## Workspace File Organization — Creative Project

Organize channel workspace files as follows:

- **brief.md** — Project brief
  - Objectives and deliverables
  - Target audience and their needs
  - Constraints: budget, timeline, brand guidelines, technical limits
  - Success metrics and acceptance criteria

- **concepts.md** — Ideas, drafts, and iterations
  - Version each concept: `### V1 — Concept Name`
  - Note rationale for each direction explored
  - Mark current working direction clearly
  - Capture discarded ideas with reasons (may revisit later)

- **feedback.md** — Review notes and revision requests
  - Date-stamp each round: `## YYYY-MM-DD Review`
  - Distinguish between mandatory changes and suggestions
  - Track resolution: addressed, deferred, declined

- **tasks.md** — Kanban board for creative tasks (Mission Control compatible)

- **status.md** — Project health and phase tracking (Mission Control compatible)

- **timeline.md** — Reverse-chronological activity log (Mission Control compatible)
  - Auto-captures significant creative events: concept iterations, feedback rounds, deliverable submissions, task moves
  - Entries: `- HH:MM — Event description` grouped under `## YYYY-MM-DD` date headers
  - Append new entries at top of current day's section
  - Use `append_timeline_event` tool to log events (also auto-logged by `move_task_card`, status changes, etc.)

- **references.md** — Inspiration and style guides
  - Mood boards, color palettes, typography choices
  - Competitive examples and benchmarks
  - Brand guidelines and asset locations

Create files as needed — not all files are required from the start. Archive completed phases to the archive/ folder.

### tasks.md — Kanban Format

Use a markdown kanban board with these columns:

```markdown
## Backlog

### Create mood board for landing page
- **id**: mc-a1b2c3
- **priority**: medium
- **tags**: design, research

## In Progress

### Draft hero section copy
- **id**: mc-d4e5f6
- **priority**: high
- **tags**: copywriting
- **started**: 2026-03-28

## Review

### Logo concept v2
- **id**: mc-g7h8i9
- **priority**: high
- **tags**: branding, design
- **started**: 2026-03-25

## Done

### Finalize project brief
- **id**: mc-j0k1l2
- **priority**: critical
- **completed**: 2026-03-20
```

Each card follows the Mission Control card format:
- **id** (required): Unique `mc-XXXXXX` identifier (auto-generated by `create_task_card`)
- **priority**: `low`, `medium`, `high`, or `critical`
- **tags**: Comma-separated labels (e.g. design, copywriting, branding)
- **started** / **completed**: ISO date timestamps (auto-set by `move_task_card`)

Use `create_task_card` and `move_task_card` tools for task management instead of manual edits.

### status.md — Project Status Format

```markdown
- **phase**: Concept Development
- **health**: green
- **updated**: 2026-03-28
- **owner**: Creative Lead

## Current Focus
- Exploring visual directions for brand refresh
- Client review scheduled for Friday

## Blockers
- Waiting on final brand guidelines from client

## Recent Milestones
- 2026-03-20: Brief approved by stakeholders
- 2026-03-15: Kickoff meeting completed
```

Health values: `green` (on track), `yellow` (at risk), `red` (blocked/behind).

### timeline.md — Activity Log

Reverse-chronological event stream capturing significant creative activity. New entries go at the top of the current day's section.

```markdown
## 2026-03-28

- 16:00 — Card mc-g7h8i9 moved to **Review** (was: In Progress) — "Logo concept v2"
- 14:30 — Concept V3 added to concepts.md — minimalist direction
- 11:00 — Feedback round 2 received — 3 mandatory changes, 2 suggestions
- 09:15 — Status health changed: green → yellow — waiting on brand guidelines

## 2026-03-27

- 17:00 — Card mc-j0k1l2 moved to **Done** (was: Review) — "Finalize project brief"
- 13:00 — Client approved concept V2 direction for hero section
- 10:30 — New card created: mc-a1b2c3 "Create mood board for landing page"
```

Events are auto-logged by `move_task_card` and status changes. Use `append_timeline_event` to manually log notable events (concept iterations, client feedback, deliverable submissions). Heartbeats can auto-append periodic entries for passive monitoring.

### plans.md — Structured Execution Plans

You can create structured plans for complex goals. Use `draft_plan` when proposing multi-step work — the user reviews and approves in Mission Control before execution begins. Pull the planning skill for the full protocol.
