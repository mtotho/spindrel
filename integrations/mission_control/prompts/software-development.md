## Workspace File Organization — Software Development

Organize channel workspace files as follows:

- **project.md** — Project overview
  - Goals, scope, success criteria
  - Tech stack and key dependencies
  - Links to repos, CI/CD, staging/prod environments
  - Team roles and contacts

- **architecture.md** — System architecture
  - High-level component diagram (describe in text/ASCII)
  - Data flow between services
  - Database schema overview and key relationships
  - API contract summaries

- **tasks.md** — Kanban board for development tasks (Mission Control compatible)

- **decisions.md** — Architecture/design decisions (ADR-style)
  - Format: `### ADR-NNN: Title` with Status, Context, Decision, Consequences
  - Record alternatives considered and why they were rejected
  - Date each decision for future reference

- **status.md** — Project health and progress (Mission Control compatible)

- **notes.md** — Meeting notes, brainstorm captures, scratch work
  - Date-stamp entries: `## YYYY-MM-DD Topic`
  - Action items in bold or checklist format

- **references.md** — External links, API docs, useful resources
  - Categorize by topic (APIs, libraries, standards, tutorials)
  - Add brief annotations explaining relevance

Create files as needed — not all files are required from the start. Archive resolved items to the archive/ folder.

### tasks.md — Kanban Format

Use a markdown kanban board with these columns:

```markdown
## Backlog

### Implement user authentication
- **id**: mc-a1b2c3
- **priority**: high
- **assignee**: backend-team
- **tags**: security, api

## In Progress

### Set up CI/CD pipeline
- **id**: mc-d4e5f6
- **priority**: high
- **assignee**: devops
- **started**: 2026-03-28

## Review

### Add input validation to forms
- **id**: mc-g7h8i9
- **priority**: medium
- **assignee**: frontend-team
- **started**: 2026-03-25

## Done

### Initialize project repository
- **id**: mc-j0k1l2
- **priority**: high
- **completed**: 2026-03-20
```

Each card follows the Mission Control card format:
- **id** (required): Unique `mc-XXXXXX` identifier (auto-generated by `create_task_card`)
- **priority**: `low`, `medium`, `high`, or `critical`
- **assignee**: Team member or team responsible
- **tags**: Comma-separated labels for categorization
- **started** / **completed**: ISO date timestamps (auto-set by `move_task_card`)

Use `create_task_card` and `move_task_card` tools for task management instead of manual edits.

### status.md — Project Status Format

```markdown
- **phase**: Development — Sprint 3
- **health**: green
- **updated**: 2026-03-28
- **owner**: Project Lead

## Current Focus
- Completing authentication module
- Setting up automated test pipeline

## Blockers
- None currently

## Recent Milestones
- 2026-03-20: Project scaffolding complete
- 2026-03-15: Architecture review approved
```

Health values: `green` (on track), `yellow` (at risk), `red` (blocked/behind).
