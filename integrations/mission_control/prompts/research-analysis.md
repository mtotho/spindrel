## Workspace File Organization — Research / Analysis

Organize channel workspace files as follows:

- **question.md** — Research questions and hypotheses
  - Primary question at the top, sub-questions as sub-headers
  - State hypotheses clearly with testable predictions
  - Track which questions are answered, open, or revised

- **sources.md** — Source list with annotations
  - Format: `### Source Title` with URL, author, date, relevance score
  - Group by topic or type (papers, articles, datasets, interviews)
  - Note credibility and potential biases

- **findings.md** — Key findings and conclusions
  - Link each finding to supporting evidence (source references)
  - Distinguish between confirmed findings and tentative conclusions
  - Note confidence level: high, medium, low

- **methodology.md** — Research approach and frameworks
  - Search strategy, inclusion/exclusion criteria
  - Analysis frameworks and evaluation rubrics
  - Limitations and assumptions

- **tasks.md** — Kanban board for research tasks (Mission Control compatible)

- **status.md** — Research progress and health (Mission Control compatible)

- **timeline.md** — Reverse-chronological activity log (Mission Control compatible)
  - Auto-captures significant research events: sources added, findings confirmed, hypotheses revised, task moves
  - Entries: `- HH:MM — Event description` grouped under `## YYYY-MM-DD` date headers
  - Append new entries at top of current day's section
  - Use `append_timeline_event` tool to log events (also auto-logged by `move_task_card`, status changes, etc.)

- **notes.md** — Reading notes, raw observations, scratch work
  - Date-stamp entries: `## YYYY-MM-DD Topic`
  - Capture quotes and page references for later citation

Create files as needed — not all files are required from the start. Archive completed investigations to the archive/ folder.

### tasks.md — Kanban Format

Use a markdown kanban board with these columns:

```markdown
## Backlog

### Review literature on distributed systems
- **id**: mc-a1b2c3
- **priority**: high
- **tags**: literature-review

## In Progress

### Analyze survey dataset
- **id**: mc-d4e5f6
- **priority**: high
- **tags**: data-analysis
- **started**: 2026-03-28

## Review

### Draft findings for Section 3
- **id**: mc-g7h8i9
- **priority**: medium
- **tags**: writing
- **started**: 2026-03-25

## Done

### Define research questions
- **id**: mc-j0k1l2
- **priority**: critical
- **completed**: 2026-03-20
```

Each card follows the Mission Control card format:
- **id** (required): Unique `mc-XXXXXX` identifier (auto-generated by `create_task_card`)
- **priority**: `low`, `medium`, `high`, or `critical`
- **tags**: Comma-separated labels (e.g. literature-review, data-analysis, writing)
- **started** / **completed**: ISO date timestamps (auto-set by `move_task_card`)

Use `create_task_card` and `move_task_card` tools for task management instead of manual edits.

### status.md — Research Status Format

```markdown
- **phase**: Data Collection
- **health**: green
- **updated**: 2026-03-28
- **owner**: Lead Researcher

## Current Focus
- Analyzing survey responses (n=450)
- Cross-referencing with existing literature

## Blockers
- Waiting on IRB approval for follow-up interviews

## Recent Milestones
- 2026-03-20: Research questions finalized
- 2026-03-15: Methodology approved
```

Health values: `green` (on track), `yellow` (at risk), `red` (blocked/behind).

### timeline.md — Activity Log

Reverse-chronological event stream capturing significant research activity. New entries go at the top of the current day's section.

```markdown
## 2026-03-28

- 15:30 — Finding confirmed: "distributed caching reduces p99 latency by 40%" (confidence: high)
- 13:00 — Card mc-d4e5f6 moved to **In Progress** (was: Backlog) — "Analyze survey dataset"
- 10:45 — New source added: "Scaling Distributed Systems" (Chen et al., 2025)
- 09:15 — Hypothesis H2 revised: narrowed scope to read-heavy workloads only

## 2026-03-27

- 16:00 — Card mc-j0k1l2 moved to **Done** (was: Review) — "Define research questions"
- 11:30 — Status health changed: yellow → green — IRB approval received
- 09:00 — Literature review: 12 new papers screened, 4 included
```

Events are auto-logged by `move_task_card` and status changes. Use `append_timeline_event` to manually log notable events (source discoveries, hypothesis changes, findings confirmed). Heartbeats can auto-append periodic entries for passive monitoring.

### plans.md — Structured Execution Plans

You can create structured plans for complex goals. Use `draft_plan` when proposing multi-step work — the user reviews and approves in Mission Control before execution begins. Pull the planning skill for the full protocol.
